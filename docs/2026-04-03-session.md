## Progress since 2026-04-03

### Amoeba (incremental)

- **Subprocess & errors** — **`run_subprocess`** in **`amoeba.subprocess`** with **`SubprocessError`** in the Amoeba exception hierarchy, so FFmpeg/FFprobe/Manim invocations share timeouts, logging, and structured failure reasons instead of ad-hoc `subprocess` calls.
- **Runtime / env** — **`load_agent_env(dotenv_paths=[...])`** supports ordered `.env` resolution (e.g. repo root then package) for consistent configuration across tools.

### Manimator (pipeline, IR, docs)

- **Logging & CLI** — Central **`configure_logging`** / **`get_logger`** with context; **`main.py`** flags such as **`--log-level`**, **`--log-json`**, **`--log-file`**, **`--run-id`**; run lifecycle aligned with Amoeba observability (**`trace_id` ≈ `run_id`**, structured events).
- **Exceptions** — Typed hierarchy in **`manimator.exceptions`** (**`ManimatorError`**, **`PipelineError`**, etc.) and **`log_exception`** integration where appropriate.
- **Per-run layout** — **`outputs/runs/<run_id>/`** with **`code/`**, **`renders/`**, **`narrated/`**, **`audio/`**, **`delivery/`**, **`manim_media/`**, and **`ir/`**; pipeline and delivery code paths respect this root vs legacy global **`outputs/`**.
- **Structured LLM outputs** — **`manimator/contracts/llm_outputs.py`** (Pydantic models for permissive LLM JSON); **planner** and **scene decomposer** use **`Agent.think_and_parse`** and map into **`SceneSpec`** / **`ScenePlan`** instead of hand-rolled JSON parsing.
- **Prompt registry** — Versioned prompts for **scene planner**, **scene decomposer**, and **code repair** under **`manimator/prompts/`** with registry getters and env-driven version keys (same pattern as intent classifier).
- **Env unification** — Package entry loads **repo + manimator `.env`** via Amoeba; stray **`load_dotenv`** / duplicate env parsing removed from agents and config modules in favor of one loader.
- **Pipeline observability** — **`log_structured`** from **`amoeba.observability`** on pipeline graph nodes (`pipeline.node.start` / `pipeline.node.completed`) and in **`main.py`** for run lifecycle (`manimator.run.start` / failed / crashed).
- **IR snapshots** — **`write_ir_bundle`** in **`manimator/ir.py`** persists a **JSON bundle** under **`outputs/runs/<run_id>/ir/`** (intent, scene plan, specs, generated code, validation, render paths, critic, transcripts, etc.) as the machine-readable record of each run.
- **IR documentation** — **`manimator/docs/ir/`** (**`README`**, **`SCHEMAS`**) describes the IR layout and schema inventory for humans and downstream tooling.
- **Knowledge-graph / semantic layer (spec)** — **`manimator/docs/kg/`** holds markdown specs for ontology, **TeachingPlan IR**, reasoning, and deployment; content was committed and mirrored on the **GitHub Wiki** (dedicated pages for KG, ontology, TeachingPlan IR, reasoning, deployment) alongside the earlier **Manimator IR** wiki pages.

### Git & external artifacts

## Amoeba (agent library)

- **Packaging** — Fixed `amoeba` install layout so the real package under `src/amoeba/` is used (removed repo-root `amoeba/__init__.py` that shadowed imports).
- **LLM primitives** — Added `completion_message_text`, `acompletion_system_user`, idempotent `load_agent_env`, extended `strip_fences` (e.g. `py` tag), and wired `LLMClient` through response parsing.
- **`Agent`** — Fixed `safe_parse_json` import; integrated with intent classifier patterns; **`StatelessMemoryAdapter`** for one-shot agents; **`InMemoryBackend`** implemented enough for recall/snapshot; **`DagestanLiveBackend`** stub for `connect()`.
- **Errors & observability** — `AmoebaError` hierarchy (`LLM*`, `JSONParseError`, `StructuredOutputError`, `ConfigurationError`), structured logging, **`acompletion_safe`** (timeouts, token budget, empty-text handling), **`async_retry_llm`**, optional **`Result`** type.
- **`LLMResponse`** — Standard shape: `text`, `tokens`, `latency_ms`, `model`, `cost`, `raw` (alias **`LLMCallResult`**); **`LLMClient.last_response`** / **`Agent.last_llm_response`**; per-call **`model=`** override.
- **Tracing** — `amoeba.observability` became a package; added **`log_trace_summary`** on logger **`amoeba.trace`** (unified JSON per logical request).
- **Dependencies** — Declared `litellm`, `pydantic`, `python-dotenv` on Amoeba as appropriate.
- **Documentation** — `amoeba/docs/GUIDE.md`, `AGENTIC.md`, `docs/README.md`, root `amoeba/README.md` (usage, gotchas, experimental areas, errors/logging).

## Manimator

- **Amoeba integration** — Workspace dependency on `amoeba`; agents use shared LLM helpers; `llm_response` / `strip_markdown_code_blocks` re-export from Amoeba for compatibility.
- **Intent classifier** — Uses **`Agent`** + **`IntentClassificationPayload`**; **`StatelessMemoryAdapter`** + **`reset_history()`** per call.
- **Versioned prompts** — `manimator/prompts/` with **`Prompt`**, `intent_classifier/v1.py` & `v2.py`, **`registry.py`** + **`get_intent_prompt`** (`INTENT_CLASSIFIER_PROMPT_VERSION`).
- **Observability** — Per-request **`new_trace_id`**, **`log_trace_summary`**, **`log_llm_event`** with `prompt_version`; optional **`MANIMATOR_METRICS_JSONL`** via **`manimator.observability.metrics`**.
- **Resilience / CI** — **`INTENT_CLASSIFIER_FALLBACK_MODEL`** (one retry on **`LLMError`**); **`MANIMATOR_DRY_RUN`** heuristic path without LLM.
- **Tests** — `manimator/tests/prompts/test_intent_classifier_cases.py` (Groq-gated regression cases + dry-run test).
- **Packaging** — Setuptools entries for `manimator.prompts`, `manimator.observability`, `manimator.tests.prompts`.
